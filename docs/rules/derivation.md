This page is normative and defines ADM semantics.

# Derivation

- Diagrams, documentation, code, and prose are derived artifacts.
- Derived artifacts MUST NOT redefine ADM semantics.
- If a derived artifact conflicts with ADM, the derived artifact is incorrect.
- Architecture changes MUST be reflected in ADM.
