This page is normative and defines ADM semantics.

# Intent

## Definition
Why the system exists; goals and quality attributes.

## Allowed Content
- Goals
- Quality attributes
- Purpose statements

## Forbidden Content
- Deployable units
- Implementation details
- Runtime topology

## Common Violations
- Mixing Intent with executable structure
- Defining technical interfaces in Intent
